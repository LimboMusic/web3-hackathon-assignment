# 搭建前端仲裁中心 Arbitration 页面

## 为什么

仲裁中心是项目“区块链味”和 Web3 治理感最强的页面，负责展示仲裁员质押、回避、证据、2/3 投票、裁决结果和仲裁日志。课堂展示时，这一页要把“不是平台客服拍脑袋，而是链上质押仲裁员投票裁决”的机制讲清楚。

现有设计稿 `ObsidianVault/02-方案设计/UI设计稿/arbitration.html` 已经定义了仲裁治理台、投票面板、席位管理、投票进度、裁决结果、仲裁员列表和日志。本 change 将其转化为 React + TypeScript + Vite 静态骨架。

## 改什么

- 新增 Arbitration 页面和路由入口。
- 复刻设计稿中的仲裁状态、证据摘要、买家/卖家票数、2/3 阈值、仲裁员席位管理、投票按钮、裁决结果和日志。
- 使用 mock 仲裁员数据展示已质押、已投票、未投票、被锁定等状态。
- 用模拟投票驱动票数变化和裁决完成反馈，不调用真实 `voteDispute`。

## 影响范围

- 影响 `frontend/` 的仲裁中心页面、投票进度组件、仲裁员表格、席位管理卡片和日志组件。
- 不修改合约、测试或部署脚本。
- 不实现真实仲裁员质押、退出、投票或举报。
