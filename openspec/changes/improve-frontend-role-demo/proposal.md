# 改进前端角色化课堂演示

## Why

当前前端 Demo 为了快速展示完整状态机，把卖家、买家、纠纷处理和仲裁投票相关操作集中放在同一演示视图中。课堂上虽然能快速点通流程，但容易让观众误解为“一个钱包可以执行所有操作”，弱化了本项目最重要的 Web3 身份边界：卖家、买家和仲裁员是不同链上地址，合约权限由 `msg.sender` 决定。

用户已确认本次前端改造优先服务课堂稳定演示。因此需要在不接入真实合约的前提下，用 Demo 账号切换器模拟 MetaMask 多账号切换，并让页面根据当前地址推导角色、显示对应业务操作。

## What Changes

- 新增前端 Demo 账号模型：卖家、买家、仲裁员 A/B/C、访客。
- 顶部钱包区域或独立演示控制台支持切换当前 Demo 账号。
- 交易详情页根据当前账号与交易数据推导当前角色：
  - 当前地址等于商品卖家：卖家视角。
  - 当前地址等于商品买家：买家视角。
  - 当前地址为 active arbiter 且不是本单买卖双方：仲裁员视角。
  - 未连接或无关地址：访客/只读视角。
- 业务主区域只展示当前角色可执行操作；其他角色操作折叠展示不可用原因。
- 新增独立“演示控制台”，支持重置或跳转到课堂剧本状态，但不混入业务按钮。
- 仲裁中心去掉“任意选择仲裁员席位作为业务身份”的主交互，改为通过当前 Demo 账号决定仲裁员身份。

## Impact

- 影响前端 mock 状态、钱包上下文、交易详情页、仲裁页、顶部状态栏和样式。
- 不修改 Solidity 合约、Hardhat 测试、部署脚本、Sepolia 部署产物。
- 不接入真实 MetaMask / Ethers.js；但实现方式应为后续真实钱包地址接入保留同一套角色推导规则。

## Non-goals

- 不实现真实链上读写。
- 不新增合约函数或改变合约权限模型。
- 不把 Demo 控制台包装成真实业务功能。
- 不引入复杂 DAO、仲裁员白名单管理或稳定币支付。
