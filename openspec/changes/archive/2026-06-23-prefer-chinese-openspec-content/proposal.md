# OpenSpec 内容优先中文

## 为什么

项目文档、报告和课堂展示主要使用中文。为了降低后续阅读和维护成本，OpenSpec 中的需求、任务和影响说明也应尽量使用中文。

## 改什么

- 在 `AGENTS.md` 中补充 OpenSpec 优先中文的规则。
- 在 `openspec/project.md` 中新增 OpenSpec 语言约定。
- 将现有 OpenSpec 文档正文尽量中文化。
- 在项目工作流规格中加入“OpenSpec 内容优先中文”的要求。

## 影响

- 后续 OpenSpec change 的 proposal、tasks 和 spec 正文会优先使用中文。
- 少量工具关键词、命令、目录名、文件名、代码标识和专有名词仍可保留英文，以保证 OpenSpec CLI 可以正常校验。
