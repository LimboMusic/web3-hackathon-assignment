# 明确前端设计稿引用与多页面实现方向

## 为什么

当前 `技术实现文档.md` 只在页首写了“前端页面参考[[UI设计稿]]”，但没有给出文件系统路径，也没有明确后续 agent 在前端开发前必须阅读设计稿。同时，步骤 9 仍写作“React/Vite 单页”，会与后续已生成的多页面 UI 设计稿方向冲突。

## 改什么

- 在技术实现文档顶部补充 `ObsidianVault/02-方案设计/UI设计稿/` 的明确路径。
- 要求执行 `add-frontend-demo` 与 `integrate-frontend-contract` 前必须先阅读该目录下已有 HTML 设计稿。
- 将步骤 9 从“单页 Demo”调整为多页面演示应用：总览、交易市场、交易详情、仲裁中心、部署演示。
- 将步骤 10 明确为在设计稿对应页面中接入 Ethers.js、MetaMask、ABI/地址、交易三态、事件监听和角色校验。

## 影响范围

- 仅修改项目规划与实施文档，不修改合约、测试、部署脚本或前端代码。
- 后续前端开发应以 `UI设计稿/` 目录中的 HTML 文件为视觉和信息架构基准。
